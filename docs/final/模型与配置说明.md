# 模型与配置说明

## 支持的大模型

### DeepSeek（默认，推荐）

| 模型 | 说明 | 推荐程度 |
|------|------|----------|
| `deepseek-v4-pro` | 最新旗舰模型，推理能力强 | ⭐⭐⭐ 推荐 |
| `deepseek-v4-flash` | 快速轻量版本 | ⭐⭐ 备选 |
| `deepseek-chat` | 旧版兼容，不推荐长期使用 | ⭐ 应急 |
| `deepseek-reasoner` | 旧版推理模型，不推荐长期使用 | ⭐ 应急 |

- **Base URL**: `https://api.deepseek.com`
- **API Key**: 在 [platform.deepseek.com](https://platform.deepseek.com) 获取
- **超时**: 默认 60 秒，可在设置页调整

### 讯飞星火（可选，不默认启用）

| 模型 | 说明 |
|------|------|
| `generalv3.5` | 通用模型（默认） |
| `generalv3` | 旧版通用 |
| `lite` | 轻量版 |
| `pro-128k` | 长上下文 |
| `max-32k` | 大模型 |
| `4.0Ultra` | 最新旗舰 |

- **Base URL**: `https://spark-api-open.xf-yun.com/v1`
- **认证**: API Password（Bearer Token）
- **默认状态**: 关闭（`SPARK_ENABLED=false`）

## 配置文件

### `backend/.env`

项目配置由 `backend/.env` 文件管理。该文件：
- **不在 GitHub 上**（已加入 `.gitignore`）
- 每个人需要自己创建（可复制 `.env.example`）
- API Key 通过前端设置页填写并自动保存到此文件

### `backend/.env.example`

模板文件，包含所有可配置项及其默认值。提交到 GitHub 供参考。

### 关键配置项

```bash
# 模型选择：deepseek 或 spark
LLM_PROVIDER=deepseek

# DeepSeek 配置
DEEPSEEK_API_KEY=            # 在前端设置页填写
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_TIMEOUT_SECONDS=60

# 星火配置（可选）
SPARK_ENABLED=false
SPARK_API_PASSWORD=          # 在前端设置页填写
SPARK_MODEL=generalv3.5
SPARK_FALLBACK_PROVIDER=deepseek
```

## Fallback 机制

当主模型调用失败时，系统会自动尝试备用模型：

- **DeepSeek 为主** → 若 Spark 已启用，自动 fallback 到 Spark
- **Spark 为主** → 自动 fallback 到 DeepSeek
- **两者都失败** → 使用 Mock 离线模式

## 安全机制

1. **API Key 隔离**: 仅保存在 `backend/.env`，不进入版本控制
2. **前端保护**: API Key 输入框为 `password` 类型，不显示明文
3. **日志脱敏**: 后端日志只打印 `key=sk-4***xxxx` 格式的掩码
4. **API 响应脱敏**: `/api/settings/status` 只返回 `configured: true/false`，不返回 Key
5. **localStorage 隔离**: API Key 不写入浏览器 localStorage

## 配置方式

### 方式一：前端设置页（推荐）

打开系统设置页面，在 DeepSeek 卡片中直接填写并保存。适合非开发者。

### 方式二：直接编辑 .env

```bash
cp backend/.env.example backend/.env
# 编辑 backend/.env 填写 API Key
```

然后重启后端。适合开发者。
