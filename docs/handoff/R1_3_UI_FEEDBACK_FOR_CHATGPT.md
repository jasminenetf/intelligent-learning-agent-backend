## P0 阻塞级问题 (5个)

### P0-1: 学习助手完全空，没有任何演示内容
- **页面**: 学习助手
- **复现**: 刷新页面→点击"学习助手"
- **表现**: 聊天空、5个Artifacts Tab空、溯源面板空、画像显示"尚未提取"
- **期望**: 打开即有欢迎消息+示例问答+预生成的mindmap/quiz/lecture
- **可能原因**: 前端没有预加载种子数据或默认示例
- **推荐修法**: app初始化时自动调用/generate预生成5类资源并填充到Artifacts；chat显示欢迎消息+3个推荐问题
- **需改**: 前端

### P0-2: 资源生成中心所有卡片和结果Tab全空
- **页面**: 资源生成中心
- **复现**: 点击"资源生成"
- **表现**: 6张卡片显示"可生成"但结果区5个Tab全空
- **期望**: 预加载当前课程的典型资源；至少思维导图Tab有预生成的Mermaid图
- **推荐修法**: 页面初始化时自动调用/api/app/generate预生成mindmap和study_plan
- **需改**: 前端

### P0-3: 一键演示完成后看不到任何可视化结果
- **页面**: 数据看板
- **复现**: 点击"一键演示"→等待完成
- **表现**: 7个步骤显示✅，但只有文字描述，没有跳转到结果
- **期望**: 演示完成后自动打开学习助手的Artifacts Tab，显示思维导图；提供一个"下载PPT"按钮
- **推荐修法**: run-demo完成后自动切换页面到学习助手，激活对应Tab
- **需改**: 前端 + 后端(run-demo返回资源ID让前端定位)

### P0-4: 课程管理只显示1门课，其他3门消失
- **页面**: 课程管理
- **复现**: 点击"课程管理"
- **表现**: 只有"高等数学上"，缺失AI基础/高等数学/高等数学
- **可能原因**: S.courses只从demo-init缓存了一门课，bootstrap异步调用未完成
- **推荐修法**: loadCourses改为先await bootstrap再渲染；或直接使用/api/courses+chunks_count增强
- **需改**: 后端(给/api/courses加chunks_count) + 前端

### P0-5: 学习路径页面依赖实时API可能超时
- **页面**: 学习路径
- **复现**: 点击"学习路径"
- **表现**: 一直显示spinner，API超时后无fallback
- **推荐修法**: 预生成缓存；加超时fallback显示规则生成的路径
- **需改**: 后端(加缓存) + 前端(加超时fallback)
