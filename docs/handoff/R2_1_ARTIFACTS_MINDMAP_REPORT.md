## 本轮目标
在现有 frontend-demo 基础上，不迁移 Next.js/React，不引入复杂构建系统，用纯 HTML/CSS/JS 实现产品级前端体验，重点是思维导图产品化、Artifacts 工作区产品化、源码折叠、图谱缩放、文献溯源卡片、多智能体过程可见。

## PDF 中采纳的设计原则
- 图形优先：默认只显示图谱，不显示源码
- 工具栏：标题/状态/操作按钮在顶部
- 信息栏：底部显示 AI 模型/资料增强/画像/引用数量
- 源码折叠：默认隐藏，按需查看
- 可缩放 Mermaid：放大/缩小/重置/适应窗口 + 拖拽平移
- CitationCardGrid：卡片式文献溯源
- Agent Trace Viewer：多智能体过程卡片
- 聊天长回答折叠：超过 600 字默认折叠

## 思维导图修复前问题
1. 图形偏小、偏左，Mermaid 源码直接暴露在视觉空间
2. 图形区域有大面积空白
3. 容器内嵌滚动条难用
4. 无缩放/适应窗口/复制源码等工具
5. 状态标签缺失，信息不完整

## Mermaid 清洗与 fallback
- `cleanMermaidSource()` 函数实现
  - 去除 ```mermaid 和 ``` 标记
  - 检测并提取多个代码块中的 Mermaid 部分
  - 验证合法开头：mindmap / graph TD / graph LR / flowchart 等
  - 非合法 Mermaid 自动使用 fallback mindmap
- 渲染失败时：显示友好提示 + 查看源码按钮，不抛 console error
- Fallback mindmap（产品级）：
```
mindmap
  root((导数与极限))
    极限基础 → 极限定义/左右极限/极限运算
    导数理解 → 瞬时变化率/切线斜率/导函数
    求导方法 → 基本公式/四则运算/复合函数
    常见误区 → 极限不存在/连续与可导混淆
    学习建议 → 先看图像/再做例题
```

## 文献溯源卡片
- 卡片式网格布局，非简单文件名列表
- 每张卡片：文件名、置信度徽章、引用片段（最多3行）、页码、查看来源按钮
- 置信度分级：高≥70%（绿色）、中≥40%（黄色）、低<40%（灰色）
- 无引用时显示："提问后将在这里展示课程资料引用"

## Agent 过程卡片
- 4 个 Agent 卡片：AI学习助手、资料检索、内容校验、资源生成
- 每个 Agent：图标、名称、状态标签、详情
- 状态：待执行/进行中/已完成/失败
- 运行时动画（pulse）
- 不显示 LangGraph/DAG 等技术词

## 聊天长回答处理
- 消息区独立滚动，输入区固定底部
- AI 回答超过 600 字默认折叠，显示渐变遮罩
- "展开完整回答 ▼" / "收起 ▲" 按钮
- citations 不放正文底部，在右侧文献溯源面板展示
- 错误提示改为结构化的错误卡片（标题/详情/建议/重试按钮）
- 不再使用"网络错误"等粗糙提示

## 资源生成中心结果区
- 结果摘要卡片：当前主题、已生成资源数、引用数量、PPT 可下载状态
- 每个 Tab 空状态统一：图标 + 标题 + "该资源尚未生成" + "⚡ 立即生成"按钮
- "生成全部 5 类资源"按钮带 7 步进度条
- 完成后 toast："已生成讲义、思维导图、测验、PPT 和学习路径"
