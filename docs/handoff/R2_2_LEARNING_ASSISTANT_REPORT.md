## 本轮目标
强化"学习助手"主流程，让它从"有内容的页面"变成"真正可用的 AI 学习助手"。核心改动：示例问题一键触发 ask、Agent 过程动画化、文献溯源联动、快捷生成基于当前问题。

## 修改清单

### 前端 app.js（约 11 处精确修改）
1. S 状态增加 `lastQuestion` 和 `lastAnswerTopic` 字段
2. 新增 `_askQuestion(text)` 函数 — 核心 ask 逻辑，含 Agent 动画、citations、回答摘要
3. 新增 `animateAgentAsk()` — Agent 4 步顺序动画（800ms 间隔）
4. 新增 `agentMarkFail()` — 失败时标记 Agent 状态
5. 新增 `buildAnswerSummaryCard(data)` — 回答后显示摘要：引用数 + 一键生成按钮
6. 新增 `buildErrorCard(detail)` — 统一结构化错误卡片
7. 新增 `highlightCite(idx)` — 点击引用高亮对应来源卡
8. 修改 `_sendQuestion` — 简化为调用 `_askQuestion`
9. 修改示例问题按钮 — `onclick="_askQuestion('...')"` 直接触发
10. 修改 `_quickGenerate` — topic 使用 `S.lastAnswerTopic`
11. 修改 `renderCitations` — 增加 [1] [2] 编号、高亮交互、无引用提示
12. 修改 `initAssistant` — 更丰富的空状态（当前课程/资料状态/推荐操作）

### 前端 app.css（2 处）
1. 新增 `.cite-card .cite-num` — 引用编号徽章样式
2. 新增 `.cite-card.highlighted` — 引用高亮样式

## Agent 过程动态化结果
测试观察到的 Agent 状态流转：
1. 800ms: AI学习助手 running → 资料检索 pending
2. 1600ms: 资料检索 running → 内容校验 pending
3. 2400ms: 内容校验 running → 资源推荐 pending
4. 回答完成后: 全部 completed（资料检索显示 N 条引用）

失败时 Agent 标记为 failed 状态 ✅

## 录屏路径
推荐录屏顺序：
1. 进入学习助手 → 查看空状态（课程/资料/推荐操作）
2. 点击示例问题 → Agent 4 步动画
3. 回答完成 → 引用卡片出现
4. 点击 [1] 引用 → 来源卡高亮
5. 展开完整回答
6. 点击"生成思维导图" → 自动切换 Tab
7. 点击回答摘要中的"生成测验" → 测验卡片显示
8. 展示 5 个 Artifacts Tab 内容
