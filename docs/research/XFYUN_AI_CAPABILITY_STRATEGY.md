## 4. 科大讯飞官网能力调研

### 4.1 星火认知大模型 (Spark LLM)

- **接口**: HTTP API, `/v1/chat/completions`, OpenAI-compatible
- **base_url**: `https://spark-api-open.xf-yun.com/v1/`
- **鉴权**: APIPassword
- **模型**: generalv3.5 / generalv4.0 等
- **本项目状态**: ✅ **已完成** — `SparkLLMProvider` 实现完整，通过 `SPARK_API_PASSWORD` 鉴权

### 4.2 语音合成 (TTS)

- **能力**: 流式文本输入 → 流式音频输出
- **超拟人语音合成**: 支持逐字输出，适合大模型朗读场景
- **限制**: 需要服务开通 + 密钥
- **本项目状态**: ❌ 未接入。配置项已预留（XFYUN_VH_VOICE_ID），`VIRTUAL_HUMAN_ENABLED=false`

### 4.3 语音听写 (ASR)

- **能力**: 实时语音转文字
- **限制**: 麦克风权限、浏览器兼容、录屏稳定性
- **本项目状态**: ❌ 未接入

### 4.4 OCR 文字识别

- **能力**: 图片/扫描件文字提取
- **价值**: 对扫描教材有潜在价值
- **本项目状态**: ❌ 未接入（OCR_OUTPUT_DIR 已预留）

### 4.5 语音评测

- **能力**: 发音评测、口语打分
- **适用场景**: 语言类课程
- **本项目状态**: ❌ 未接入（不适用 AI 导论主线）

### 4.6 数字人/虚拟人

- **能力**: 虚拟形象 + 语音合成 + 口型同步
- **限制**: 需商业授权、API 权限、视频链路复杂
- **本项目状态**: ⚠️ 配置项已预留，`VIRTUAL_HUMAN_ENABLED=false`，不做真实接入

### 4.7 翻译/多语言

- **能力**: 多语种互译
- **本项目状态**: 未来扩展

## 7. 为什么不做完整数字人

完整数字人看起来很炫，但对当前比赛是净负收益：

```
1. 密钥与成本：需要真实 API 权限、服务开通、可能需购买额度。
2. 稳定性风险：视频生成/流式播放链路远不如纯文本稳定，
   会引入播放器兼容、加载状态、失败降级、网络延迟等新 bug。
3. 演示干扰：7 分钟演示里数字人会抢走"学习闭环"主线。
   评委更看重画像 → 资源 → 讲解 → 测验 → 路径 → 评估的完整链路。
4. 开发成本：至少 3-4 个子系统（视频合成 + 播放器 + 流控 + 降级），
   在当前时间窗口内不可控。
5. 可信度：说"我们有数字人"但只有一个静态卡片，
   比说"我们预留了数字人入口并规划了落地路径"更不可信。
```

---

## 8. 为什么不切 Spark 为默认模型

```
1. 无真实凭证：当前环境无 SPARK_API_PASSWORD 真实密钥，
   切换默认模型后整个系统将不可用。
2. 演示稳定性：DeepSeek 已验证稳定，E2E 通过。
   切换未验证模型是最大的演示风险。
3. 合规性：讯飞 API 密钥不应出现在比赛提交的代码仓库中。
4. 策略更优：SparkProvider 作为"已接入、可切换的国产模型"
   比"默认使用但没验证过的国产模型"更有说服力。
```

答辩话术：
> "系统已完整接入科大讯飞星火大模型，支持 APIPassword 一键切换。
> 当前比赛演示为保证稳定，默认使用 DeepSeek；
> 模型层支持 Spark → DeepSeek → Mock 三层 fallback，
> 根据配置和可用性自动选择最优模型。"

## 11. 答辩话术

### 关于国产模型

> 本项目已完整接入科大讯飞星火大模型，通过 OpenAI-compatible HTTP 接口实现。
> 系统支持通过 APIPassword 一键切换至星火模型，模型层设计为
> Spark → DeepSeek → Mock 三层自动 fallback。
> 当前比赛演示为保证端到端稳定，默认使用 DeepSeek；
> 但所有代码、配置、测试均已就绪，切换只需一条配置项。

### 关于数字人/多模态

> 数字人和语音交互方向我们采用轻量化策略：
> 系统预留了课堂助手入口，并能基于当前回答、引用来源和学习路径
> 生成微课讲解稿；后续可接入科大讯飞语音合成或数字人服务，
> 将讲解稿转化为语音或数字人微课。
> 语音合成、OCR、数字人接口配置已在系统中预置，属于明确的技术储备。

### 关于未来规划

> 我们的技术路线是：
> 第一步（已交付）：星火大模型可配置接入
> 第二步（当前可选）：微课讲解稿，为语音/数字人提供前置内容
> 第三步（未来）：TTS 语音合成 + OCR 教材扫描
> 第四步（远期）：完整数字人教学助手
